# Sitewide Copy Clarity Guide

## Goal
Make every core message answer three questions fast:
- What is going wrong?
- Who is this for?
- What should they do next?

## Voice Rules
- Speak to observable problems, not vague feelings.
- Name cross-team friction directly: product, engineering, operations, data, architecture.
- Keep one clear CTA per section.
- Prefer "delivery is blocked/stalling" over generic phrases like "momentum" or "slipping releases."

## ICP Framing
- Founders and entrepreneurs: roadmap commitments slip after growth.
- Operators and product leaders: dependencies, ownership, and integration work block delivery.
- Enterprise teams: architecture and governance friction slows execution.

## Executed Route Updates
- Global metadata and social copy now lead with clear delivery bottlenecks.
- Homepage hero now names missed commitments and system bottlenecks directly.
- Services hero now asks a concrete question and uses gradient emphasis on the solution.
- Launchpad hero now says "product delivery" instead of "releases."
- Contact and Founder Review copy now references delivery bottlenecks clearly.
- Footer copy now speaks to founders, operators, and enterprise teams.

## Conversion Rules Going Forward
- Hero formula:
  - Problem question
  - Plain-language solution line
  - One primary CTA
- Section intros:
  - "When X is happening, we do Y so you can do Z."
- Avoid:
  - "working hard but"
  - "regain momentum"
  - "clarity and outcomes" without context
